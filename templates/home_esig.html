Basic examples

AK test

1. Request a signature through your app (embedded signing)

Sends an envelope, then uses embedded signing for the first signer. With embedded signing, DocuSign signing is initiated from your website.

API methods used: Envelopes::create, EnvelopeViews::createRecipient.

2. Request a signature by email (remote signing)

The envelope includes three documents: one each in PDF, Word, and HTML format. Anchor text is used to position the signing fields in the documents.

API method used: Envelopes::create.

3. List envelope status changes

Lists the envelopes whose status has changed within a specified range of dates. The envelopes' current status is included.

API method used: Envelopes::listStatusChanges.

4. Get envelope information

Lists basic information about an envelope, including its overall status.

API method used: Envelopes::get.

5. List envelope recipients

Lists the envelope's recipients, including their current status.

API method used: EnvelopeRecipients::list.

6. List envelope documents

Lists the envelope's documents. A Certificate of Completion is included with every envelope.

API method used: EnvelopeDocuments::list.

7. Download envelope documents

Downloads individual documents, the documents concatenated together, or a ZIP file of the documents.

API method used: EnvelopeDocuments::get.

8. Create a template

Creates a template with two roles, signer and cc recipient.

API methods used: Templates::list, Templates::create.

9. Request a signature by email using a template

The envelope is defined by the template. The signer and cc recipient name and email are used to fill the template's roles.

API method used: Envelopes::create.

10. Attach documents using binary transfer

Multipart data transfer is used to send the documents in binary format to DocuSign. Binary transfer is 33% more efficient than Base64 encoding.

API method used: Envelopes::create.

11. Send an envelope in your app (embedded sending)

Embeds the DocuSign UI in your web app to finalize or update the envelope and documents before they are sent.

API methods used: Envelopes::create, EnvelopeViews::createSender.

12. Embed the DocuSign UI in your app

Use this API call to open the DocuSign UI with the user already logged in.

API method used: EnvelopeViews::createConsole.

13. Request a signature using a composite template

Sends an envelope based on a template. In addition to the template's document, this example adds an additional document to the envelope using the Composite templates feature.

API methods used: Envelopes::create, EnvelopeViews::createRecipient.

Payments

14. Send a request for payment

Sends an order form with online payment by credit card.

API method used: Envelopes::create.

Tabs

15. Get envelope tab values

Retrieves the tab (field) values for all of the envelope's recipients.

API method used: EnvelopeFormData::get.

16. Set envelope tab values

Creates an envelope and sets the initial values for its tabs (fields). Some of the tabs are set to read-only, others can be updated by the recipient. This example also stores metadata with the envelope.

API methods used: Envelopes::create, EnvelopeViews::createRecipient.

17. Set tab values in a template

Creates an envelope using a template and sets the initial values for its tabs (fields). This example also stores metadata with the envelope.

API methods used: Envelopes::create, EnvelopeViews::createRecipient.

18. Get envelope custom tab values

Retrieves the custom metadata (custom data fields) stored with the envelope.

API method used: EnvelopeCustomFields::list.

Recipient authentication

19. Require access code authentication for a recipient

Sends an envelope that requires entering an access code for the purpose of multifactor authentication.

API method used: Envelopes::create.

20. Require phone authentication for a recipient

Sends an envelope that requires entering a six-digit code from a text message or phone call for the purpose of multifactor authentication.

API method used: Envelopes::create.

22. Require knowledge-based authentication (KBA) for a recipient

Sends an envelope that requires passing a public records check to validate identity for the purpose of multifactor authentication.

API method used: Envelopes::create.

23. Require ID verification (IDV) for a recipient

Sends an envelope that requires the recipient to upload a government-issued ID for the purpose of multifactor authentication.

API methods used: Accounts::getAccountIdentityVerificationsList, Envelopes::create.

Permissions

24. Create a permission profile

Creates a user group’s permission profile.

API method used: AccountPermissionProfiles::create.

25. Set a permission profile

Sets a user group's permission profile. You must have already created a permission profile and a group of users.

API method used: Groups::update.

26. Update individual permission settings

Edits individual permission settings on a permission profile.

API method used: AccountPermissionProfiles::update.

27. Delete a permission profile

Lists all permission profiles and allows you to delete any without associated users. Note: You cannot remove "Everyone" nor "Administrator" permission profiles.

API method used: AccountPermissionProfiles::delete.

Brands

28. Create a brand

Creates a brand profile for an account.

API method used: AccountBrands::create.

29. Apply a brand to an envelope

Applies a brand you've created to an envelope. Anchor text is used to position the signing fields in the documents.

API method used: Envelopes::create.

30. Apply a brand and template to an envelope

Creates an envelope from a template and applies your selected brand. You must have at least one created template and brand. Anchor text is used to position the signing fields in the documents.

API method used: Envelopes::create.

Bulk operations

31. Bulk send envelopes

Sends envelopes in bulk to multiple recipients. First, this example creates a bulk send recipients list, then creates an envelope. After that, it initiates bulk envelope sending.

API methods used: BulkSend::createBulkSendList, Envelopes::create, EnvelopeCustomFields::create, BulkSend::createBulkSendRequest, BulkSend::getBulkSendBatchStatus

Advanced recipient routing

32. Pause a signature workflow

Creates an envelope where the workflow is paused before the envelope is sent to a second recipient.

API method used: Envelopes::create.

33. Unpause a signature workflow

Updates an envelope to resume the workflow that has been paused. You must have created an envelope with a paused signature workflow to run this example.

API method used: Envelopes::update.

34. Use conditional recipients

Creates an envelope where the workflow is routed to different recipients based on the value of a transaction.

API method used: Envelopes::create.

Premium features

35. Request a signature by SMS delivery

Sends a signature request via an SMS message.

API method used: Envelopes::create.
